# 99 총정리

*네트워크 면접 대비*



## 1. 네트워크의 이해와 설정

1. 네트워크란 무엇인가요?

   ```
   [Keyword] : 컴퓨터, 연결, 통신
   
   - 네트워크란, 두 대 이상의 컴퓨터들을 연결하고 서로 통신할 수 있도록 하는 데이터 통신 체계입니다.
   ```

2. 인터넷, 인트라넷, 엑스트라넷의 차이점은 무엇인가요?

   ```
   [Keyword] : 사용자 엑세스 수준
   
   - 인터넷, 인트라넷, 엑스트라넷은 공통적으로 유사한 TCP/IP 기술을 사용하지만, 네트워크 내부와 외부의 각 사용자에 대한 엑세스 수준이 다릅니다.
   
   [인터넷]
   - 인터넷은 웹을 사용하는 모든 위치에서 누구나 응용 프로그램에 엑세스할 수 있습니다.
   
   [인트라넷]
   - 인트라넷은 동일한 조직의 사용자에게 제한된 접근만 허용합니다.
   
   [엑스트라넷]
   - 엑스트라넷은 외부 사용자에게 조직의 네트워크 응용 프로그램을 사용할 수 있는 액세스 권한이 부여됩니다.
   ```

## 2. 네트워크의 구성

1. 스위치와 허브의 차이점을 말해보세요.

   ```
   [Keyword] : 전송 기기 선택
   
   - 가장 큰 차이점은 연결된 기기들 중에서 선택하여 전송할 수 있는지 여부입니다.
   - 허브는 하나에서 전송된 패킷이 허브에 연결된 모든 기기로 브로드캐스팅됩니다.
   - 반면, 스위치는 패킷의 목적지 주소로 지정된 포트로만 패킷이 전달됩니다.
   - 또한, 스위치는 동시에 송수신 할 수 있으므로 허브보다 빠른 속도로 데이터를 보냅니다.
   ```

2. 127.0.0.1과 localhost는 무엇을 의미합니까?

   ```
   [Keyword] : local IP 주소
   
   - 둘 다 PC 자신의 local IP 주소를 의미합니다.
   - 보통 연결 문제를 식별하기 위해 ping을 통해 응답하는지 확인하는 용도로 쓰입니다.
   - 이때, 127.0.0.1은 원래 IP 주소라면, localhost는 127.0.0.1의 DNS라고 볼 수 있습니다.
   - 그러나 localhost에 127.0.0.1을 연결하지 않을 수도 있습니다.
   ```

3. 네트워크에는 어떤 유형이 있나요?

   ```
   [Keyword] : PAN, LAN, MAN, WAN
   
   [PAN(개인 영역 네트워크)]
   - 가정에서 사용하는 가장 작고 기본적인 네트워크 유형입니다.
   - 컴퓨터와 전화, 프린터 등과 같은 다른 장치들을 연결합니다.
   
   [LAN(근거리 통신망)]
   - 사무실 등 소규모 컴퓨터 그룹을 서로 연결하는 데 사용되는 네트워크 유형입니다.
   
   [MAN(Metropolitan Area Network)]
   - LAN보다는 넓고 WAN 보다는 작은 지역 간 연결하는데 사용되는 네트워크 유형입니다.
   
   [WAN(광역 네트워크)]
   - LAN보다 복잡하며 물리적 거리가 넓은 영역을 포함하는 네트워크 유형입니다.
   - 인터넷
   ```

## 3. 네트워크 통신

1. 무상태와 비연결성에 대해 설명해보세요.

   ```
   [Keyword] : 요청과 응답, 상태 유지하지 않음, HTTP, 쿠키/세션
   
   - 비연결성이란, 클라이언트가 보낸 요청에 대해 서버가 응답을 한 후 바로 연결이 끊어지는 특징을 의미합니다.
   - 무상태성은 이러한 비연결적인 특징을 바탕으로 연결이 끊어지며 요청한 결과에 대해 잊어버려 상태를 유지하지 않는 특징을 말합니다.
   - HTTP는 비연결성, 무상태의 특징을 가지는데 상태를 유지하기 위해서 쿠기 또는 세션을 이용할 수 있습니다.
   
   <무상태성, 비연결성의 장단점>
   *장점: 서버 입장에선 통신 연결을 유지하지 않아 리소스 낭비가 줄어 듦
   *단점: 통신마다 새로운 세션을 열어야 하는 작업은 클라이언트가 서버에게 요청 시마다 인증해야 하는 단점
   ```

2. 쿠키와 세션의 차이

   ```
   [Keyword] : 정보 저장 위치, 정보 특성
   
   - 쿠키와 세션의 가장 큰 차이점은 정보가 저장되는 위치입니다.
   - 쿠키는 클라이언트 측에, 세션은 서버 측에 저장되고 관리됩니다.
   - 정보의 특성에 따라, 주로 가로채기 당하거나 했을 때 큰 문제가 생기지 않을 만한 정보들을 쿠키에, 문제가 생길만한 정보들은 세션에 저장합니다.
   ```

   

## 4. OSI 참조 모델

1. OSI 참조 모델의 계층을 각각 간략하게 설명해보세요.

   ```
   [Keyword] : 서로 다른 시스템 통신, 통신 규약, 물리, 데이터링크, 네트워크, 전송, 세션, 표현, 응용
   
   - OSI 참조 모델은 다른 시스템 간의 원활한 통신을 위해 ISO에서 제안한 통신 규약입니다.
   [OSI 7계층]
   - 7 계층(응용 계층): 사용자와 직접 상호작용하는 응용 프로그램들이 포함된 계층
   - 6 계층(표현 계층): 데이터의 형식(Format)을 정의하는 계층
   - 5 계층(세션 계층): 컴퓨터끼리 통신을 하기 위해 세션을 만드는 계층
   - 4 계층(전송 계층): 최종 수신 프로세스로 데이터의 전송을 담당하는 계층
   - 3 계층(네트워크 계층): 패킷을 목적지까지 가장 빠른 길로 전송하기 위한 계층
   - 2 계층(데이터링크 계층): 데이터의 물리적인 전송과 에러 검출, 흐름 제어를 담당하는 계층
   - 1 계층(물리 계층): 데이터를 전기 신호로 바꾸어주는 계층
   ```

2. DNS에 대해 설명해보세요.

   ```
   [Keyword] : IP주소, 이름
   
   [DNS (Domain Name Server)]
   - DNS란 IP주소를 그대로 기억하기 어려워 나온 개념으로, IP주소에 붙인 이름입니다.
   
   Domain Name System은 호스트의 도메인 이름을 호스트의 네트워크 주소로 바꾸거나 그 반대의 변환을 수행할 수 있도록 하기 위해 개발되었다. 특정 네트워크의 주소를 찾기 위해 사람이 이해하기 쉬운 도메인 이름을 숫자로 된 식별 번호(IP주소)로 변환해 준다.
   ```

## 5. 네트워크 계층(IP)

1. 프로토콜이란 무엇인가요?

   ```
   [Keyword] : 데이터 교환, 표준화, 통신 규약
   
   - 프로토콜이란 서로 다른 기기들 간의 데이터 교환을 원활하게 수행할 수 있도록 표준화해놓은 통신 규약입니다.
   
   * 대표 5가지 기능
   	- 단편화와 재결합 : 데이터를 단편화 전송함으로써 빠르고 오류 제어하기 쉬움
   	- 캡슐화 : 정보 데이터에서 오류가 생기는 것을 막기 위해 캡슐화를 해서 보냄
   	- 흐름 제어 : 수신 측이나 송신 측에서 움직이는 데이터 양이나 속도를 조절
   	- 오류 제어 : 전송 중 발생하는 오류 검출하고 정정하여 정보의 신뢰성 높임
   	- 동기화 : 송, 수신측이 같은 상태를 유지하도록 타이밍을 맞춤
   ```

2. ARP와 RARP를 비교해보세요.

   ```
   [Keyword] : IP 주소, 물리적 주소
   
   ARP (Address Resolution Protocol)
   - IP주소를 물리적 주소(MAC)로 대응시켜 주는 프로토콜
   
   RARP (Reverse Address Resolution Protocol)
   - 물리적 주소(MAC)로 IP주소를 알려주는 프로토콜
   ```

3. 데이터 캡슐화란 무엇입니까?

   ```
   [Keyword] : 계층, 헤더, 호환성, 독립성
   
   - 송신 측에서 데이터를 생성할 때 각 계층에서 헤더를 붙이며 생성하는데, 이렇게 헤더를 붙이는 작업을 캡슐화라고 부릅니다.
   - 반대로 수신 측에서 데이터를 받으며 각 계층에서 헤더를 제거하는 작업을 역캡슐화라고 합니다.
   
   <캡슐화, 역캡슐화를 하는 이유는 무엇인가?>
   - 이렇게 캡슐화와 역캡슐화를 하는 이유는 호환성 유지, 독립성 유지를 위함입니다.
   - 호환성 유지는 다른 네트워크의 동일 계층 간에 호환성을 유지하는 것을 의미하고,
   - 독립성 유지는 각 계층 간 기능을 분리함으로써 계층 간의 독립성을 유지하는 것을 의미합니다.
   ```

4. NIC란 무엇입니까?

   ```
   [Keyword] : LAN, 컴퓨터에 설치, 장치 어댑터
   
   - NIC는 Network Interface Card의 약자로, LAN에 연결 지점을 제공하기 위해 컴퓨터에 설치하는 장치 어댑터입니다.
   ```

5. 공인 IP와 사설 IP의 차이점을 설명해주세요.

   ```
   [Keyword] : 공인 IP - ISP, 공개, 보안 설정 / 사설 IP - 네트워크, 라우터, 가상 주소
   
   [공인 IP]
   - 공인 IP는 전 세계에서 유일한 IP로 ISP(인터넷 서비스 공급자)가 제공하는 IP주소입니다.
   - 외부에 공개되어 있기 때문에 인터넷에 연결된 다른 장비로부터 접근이 가능하며, 이에 따라 방화벽과 같은 보안 설정을 해줘야 한다는 특징이 있습니다.
   
   [사설 IP]
   - 사설 IP는 어떤 네트워크 안에서 사용되는 IP 주소입니다.
   - 주소 크기 제약으로 인해 모든 네트워크가 공인 IP를 사용하는 것이 불가능하기 때문에 네트워크 안에서 라우터를 통해 할당받는 가상의 주소입니다.
   ```

6. 라우팅이 무엇인지 설명하고, 라우팅 테이블을 포함하여 라우팅 과정을 설명해보세요.

   ```
   [Keyword] : 최적의 경로 선택, 경로 정보
   
   - 라우팅이란, 어떤 네트워크 안에서 데이터를 보낼 때 최적의 경로를 선택하는 과정을 말합니다.
   - 라우팅 테이블이란, 현재 네트워크에서 다른 네트워크로 데이터를 전송할 때의 경로 정보가 등록되어 있는 테이블입니다.
   ```

## 6. 전송 계층(TCP)

1. TCP와 UDP의 차이에 대해 설명해보세요.

   ```
   [Keyword] : TCP - 신뢰성, 연결지향형, 3-way 핸드셰이킹 / UDP - 비연결형, 빠른 전송 속도
   
   [TCP (Transmission Control Protocol, 전송제어 프로토콜)]
   - TCP는 신뢰성 있는 데이터를 전송하기 위해 설계된 연결지향형 프로토콜로, 순차적인 전달을 한다는 특징을 갖습니다.
   - 패킷, 흐름 제어, 단편화, 전송 보장 기능을 제공합니다.
   - 신뢰적인 연결을 위해 3-way 핸드셰이킹이 진행됩니다.
   
   [UDP (User Datagram Protocol, 사용자 데이터그램 프로토콜)]
   - UDP는 비연결형 프로토콜로, 순서를 보장하지 않으나 전송 속도가 빠르다는 특징이 있습니다.
   - 상위 계층에 신뢰성을 보장하는 프로토콜이 있거나, 멀티미디어 데이터 전송처럼 전송 속도가 중요할 때 주로 쓰입니다.
   ```

2. 3-Way-HandShaking에 대해 설명해보세요.

   ```
   [Keyword] : 데이터 전송, 신뢰, 패킷 요청
   
   - 3-way 핸드셰이킹이란 데이터를 전송하기 전에 신뢰할 수 있는 연결을 확립하기 위해 패킷 요청을 세 번 교환하는 것입니다.
   ```

## 7. 응용 계층

1. 주요 프로토콜에 대해 간략하게 설명하세요. (HTTP, FTP, SMTP, POP, SNMP, DHCP, TCP, UDP, IP, ARP, RARP, ICMP)

   ```
   HTTP (HyperText Transfer Protocol)
   - 하이퍼텍스트 문서를 교환하기 위한 프로토콜
   * 하이퍼텍스트 : 한 문서에서 다른 문서로 즉시 접근할 수 있는 텍스트
   
   FTP (File Transfer Protocol)
   - 파일 전송을 위한 프로토콜
   
   SMTP (Simple Mail Transfer Protocol)
   - 메일을 송신하기 위한 프로토콜
   
   POP (Post Office Protocol)
   - 메일을 수신하기 위한 프로토콜
   
   SNMP (Simple Network Management Protocol)
   - 네트워크 장비 요소 간에 네트워크 관리 및 전송을 위한 프로토콜
   
   DHCP (Dynamic Host Configuration Protocol)
   - IP주소를 자동으로 할당하고 관리하는 프로토콜
   
   TCP (Transmission Control Protocol)
   - 데이터 흐름을 제어하고 에러 유무를 검사하는 프로토콜
   
   UDP (User Datagram Protocol)
   - 데이터그램(데이터 전송 단위) 전송을 위한 프로토콜
   
   ARP (Address Resolution Protocol)
   - IP주소를 물리적 주소(MAC)로 대응시켜 주는 프로토콜
   
   RARP (Reverse Address Resolution Protocol)
   - 물리적 주소(MAC)로 IP주소를 알려주는 프로토콜
   
   ICMP (Internet Control Message Protocol)
   - 통신 중에 발생하는 오류 처리와 정보 경로 변경 등을 위한 제어 메시지 관리 프로토콜
   ```

2. DHCP에 대해, 동작 과정을 포함하여 설명해주세요.

   ```
   [Keyword] : IP 주소, 자동 할당, 시간, 오류
   
   [DHCP (Dynamic Host Configuration Protocol)]
   - IP주소를 자동으로 할당하고 관리하는 프로토콜입니다.
   - DHCP를 사용하면 IP 주소를 관리하는 시간을 줄일 수 있을 뿐만 아니라, IP 주소 할당 과정에서 발생하는 잠재적인 오류도 줄일 수 있습니다.
   
   [DHCP 동작 과정]
   1. 서브넷에 연결되어 있는 각 DHCP 서버는 사용 가능한 IP 주소를 클라이언트 컴퓨터에 동시에 보내줌
   ```

2. HTTP와 HTTPS 통신 방식의 차이에 대해 설명해보세요. (암호화되지 않은 프로토콜 vs 암호화된 프로토콜)

   ```
   [Keyword] : 
   
   [HTTP]
   - HTTP는 평문 통신이기 때문에 도청이 가능하고, 통신 상대를 확인하지 않아 위장이 가능하며, 완전성을 증명할 수 없어 변조가 가능하다는 특징이 있습니다.
   - TCP와 직접 통신하는 형태로 진행됩니다.
   
   [HTTPS]
   - 이러한 HTTP의 보안적 취약점을 보완하기 위해 나온 것이 HTTPS입니다.
   - HTTPS는 HTTP의 통신하는 소켓 부분을 SSL이라는 프로토콜로 대체한 형태의 프로토콜입니다.
   - HTTPS에서는 TCP와 직접 통신하지 않고 SSL을 거쳐서 통신하는 형태로 진행됩니다.
   - 이때, SSL을 통해 암호화, 증명서 등을 통해 보완성을 강화합니다.
   ```

3. 그렇다면, 무조건 HTTPS를 쓰는게 좋은지?

   ```
   [Keyword] : 보안 문제 크리티컬함, 최근 속도 차이 없음
   
   - 보완적인 문제는 매우 중요한 부분이기 때문에 최근 HTTPS를 적용하는 방향으로 바뀌어가고 있습니다.
   - 이전에는 HTTP에 비해 HTTPS가 리소스가 추가적으로 들어 속도가 저하되었으나, 최근에는 하드웨어의 발달로 속도 저하가 거의 일어나지 않고 있는 것으로 알고 있습니다.
   - 따라서 HTTP보다는 HTTPS를 쓰는 것이 더 좋다고 생각합니다.
   ```

3. GET과 POST 방식을 비교해주세요.

   ```
   [Keyword] : HTTP 프로토콜, 서버 요청, HTTP Request Message, url, Body, 데이터 크기, 보안성, 조회, 변경
   
   - GET과 POST는 둘 다 HTTP 프로토콜을 이용해서 서버에 요청할 때 사용하는 방식입니다.
   - GET의 경우, 요청하는 데이터가 HTTP Request Message의 Header 부분에 url 안에 담겨서 전송됩니다. 이때 url에 담겨서 전송되기 때문에, 데이터의 크기가 제한적이고 보안성이 떨어집니다.
   - 이에 따라 보통 서버에서 데이터를 가져와서 보여주는 용도로 사용됩니다.
   - POST의 경우, 요청하는 데이터가 HTTP Request Message의 Body 부분에 데이터가 담겨서 전송됩니다. 따라서 담을 수 있는 데이터 크기가 크고 암호화가 가능하며 보안성이 높습니다.
   - 중요하게 서버 DB의 값을 변경할 때 주로 사용됩니다.
   ```
