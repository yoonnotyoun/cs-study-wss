## 심화문제

1. 

   ```
   점유와 대기 예방
   - 프로세스가 자신이 사용하는 모든 자원을 자세히 알기 어렵다
   - 자원의 활용성이 떨어진다
   - 많은 자원을 사용하는 프로세스가 적은 자원을 사용하는 프로세스보다 불리하다
   - 결국 일괄 작업 방식으로 동작한다
   ```

2. 

   ```
   - 프로세스가 자신이 사용할 모든 자원을 미리 선언해야 한다
   - 시스템의 전체 자원 수가 고정적이어야 한다
   - 자원이 낭비된다
   ```
   
3. 

   ```
   장점: 특별한 알고리즘이 없어 쉽게 구현 가능
   단점:
   - 엉뚱한 프로세스가 강제 종료될 수 있다
   - 모든 시스템에 적용할 수 없다
   ```
